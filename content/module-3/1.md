## From chatbot to agent

A standard chatbot takes your message and returns a response. One input, one output. Done.

An agent is different. It takes a goal and pursues it — deciding what steps to take, using tools to act, observing the results, and continuing until the goal is met or it gets stuck.

## The agent loop

Every agent runs some version of this loop:

1. **Perceive** — receive input (your instruction, or output from the previous step)
2. **Plan** — decide what action to take next
3. **Act** — call a tool, write a file, search the web, run code
4. **Observe** — read the result
5. **Loop** — go back to step 1 with the new information

This continues until the task is complete, the model runs out of context, or it hits an error it can't resolve.

## What are tools?

Tools are functions the AI can call. In Claude Code, the tools include: read a file, write a file, run a bash command, search the web, ask the user a question.

The AI doesn't execute code directly. It describes what it wants to do, the framework executes it, and the result comes back into context.

## Memory: what agents remember

Agents have four types of memory:
1. **In-context** — everything in the current conversation window
2. **External** — a database or file the agent can read/write
3. **Cached** — pre-computed context stored between calls
4. **None** — most agents forget everything when the session ends, unless you build storage in

## Autonomy level

Agents exist on a spectrum. At one end: a chatbot that answers questions. At the other: a fully autonomous system that manages its own workflow, reads its own errors, and attempts to fix them.

Claude Code is a practical middle point — it can take multi-step actions, but it asks for confirmation before anything irreversible.

## Key takeaway

An agent is an AI that acts in a loop, using tools to pursue a goal. Understanding the loop, the tools, and the memory constraints tells you exactly what an agent can and cannot do.
