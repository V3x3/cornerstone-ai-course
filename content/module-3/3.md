## The problem: AI in a box

A base language model knows a lot but can do little. It can't check today's weather, search a database, run a calculation on live data, or send an email. It's knowledgeable but isolated.

Plugins and MCP servers are how you give AI hands.

## What is MCP?

MCP stands for **Model Context Protocol** — an open standard developed by Anthropic for connecting AI models to external tools and data sources.

Think of it as a universal adapter. Instead of every AI tool needing custom integrations for every external service, MCP defines a standard way for:
- AI models to call external tools
- External tools to return structured results
- The whole system to be composable and auditable

## What can MCP servers do?

An MCP server exposes a set of tools an AI can call. Examples:

| MCP Server | What it enables |
|------------|-----------------|
| Filesystem | Read/write files on your computer |
| Web search | Live internet search |
| GitHub | Read repos, create issues, submit PRs |
| PostgreSQL | Query a real database |
| Slack | Send messages, read channels |
| Browser | Visit URLs, extract page content |

## How it works in practice

You configure which MCP servers Claude has access to. When you give Claude a task, it can call those tools as needed.

Example: "Research the top 5 competitors in our market and summarise their positioning" — with a web search MCP, Claude can actually do this. Without it, it's limited to training data.

## Why this matters

MCP is the architecture that turns an AI assistant into an AI agent. The assistant knows things. The agent does things.

As more MCP servers are built (for CRMs, databases, APIs, operating systems), the range of tasks an AI can tackle autonomously expands dramatically.

## Key takeaway

MCP is how AI connects to the real world. Understanding it tells you how to extend Claude beyond conversation — into action. The tools available to an agent define what the agent can do.
