## What is a token?

A token is the unit of text that language models process. Roughly:
- 1 token ≈ ¾ of a word (in English)
- 100 tokens ≈ 75 words
- 1,000 tokens ≈ 750 words (about a page)

Common token counts:
- A short email: ~200 tokens
- A 10-page report: ~7,500 tokens
- A full novel: ~150,000 tokens

## What is a context window?

The context window is the maximum number of tokens a model can process in a single call — both input and output combined.

**Why it matters:** The model can only "see" what's in the context window. If your document is longer than the window, you have to truncate, chunk, or summarise.

Model context windows (approximate, as of early 2026):
- Claude 3.5 Sonnet: 200,000 tokens (~150,000 words)
- GPT-4o: 128,000 tokens (~96,000 words)
- Gemini 1.5 Pro: 1,000,000 tokens (exceptional outlier)

## How costs work

Most AI APIs charge per token — separately for input and output.

Example pricing (illustrative, check current rates):
- Claude Sonnet: ~$3 per million input tokens, ~$15 per million output tokens
- For a 1,000-word request + 500-word response: roughly $0.004

For personal use, these costs are negligible. For building a product that serves thousands of users, token costs become a real engineering constraint.

## Why token economics shape design decisions

When you build an AI workflow, you're constantly making token tradeoffs:

- **Shorter prompts** → cheaper, faster, less accurate
- **Longer context** → more capable, more expensive, slower
- **Caching** → reuse expensive context, save costs
- **Chunking** → process large documents in pieces, lose some coherence

Understanding this helps you design workflows that are both capable and economical.

## Key takeaway

Tokens are the currency of AI. Know roughly how many you're spending, understand the context window limit of your model, and design accordingly. This is the literacy that separates users from builders.
