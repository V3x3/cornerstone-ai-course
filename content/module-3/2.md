## Why the terminal?

The terminal is where power users work. It's direct, programmable, and composable. When you run Claude from the terminal, you're not clicking through a UI — you're giving instructions that can be chained, automated, and extended.

Claude Code is Anthropic's CLI (command-line interface) for Claude. It turns Claude into an agent that can read and write files, run commands, and work through multi-step tasks — all from your terminal.

## Prerequisites

- A terminal (Mac: Terminal or iTerm2; Windows: PowerShell or WSL; Linux: any shell)
- Node.js installed (check with `node --version`)
- An Anthropic API key (from console.anthropic.com)

## Installing Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

Set your API key:
```bash
export ANTHROPIC_API_KEY=your-key-here
```

(Add this to your shell profile — `~/.zshrc` or `~/.bashrc` — to avoid setting it every session.)

## Starting your first session

Navigate to a folder you want to work in:
```bash
cd ~/Documents/my-project
claude
```

Claude Code will start. You'll see a prompt. Type a task in plain English.

Try: "What files are in this directory and what do they do?"

Claude will use the `ls` and potentially `cat` tools to read your files and explain them.

## Key things to know

**You're in charge.** Claude asks before taking irreversible actions. If it's about to delete a file, it will tell you.

**Context is the current folder.** Claude sees your project. It can read any file, run any command you permit.

**Conversation is stateful.** Like a chat, Claude remembers what you've discussed in the session. You can build on previous instructions.

**Exit anytime.** Type `exit` or press Ctrl+C.

## A useful first task

Open Claude Code in a folder with documents or code. Ask:

"Read all the files here and give me a one-paragraph summary of what this project is."

That's it. You've just run an AI agent from the terminal.

## Key takeaway

Claude Code is Claude in your terminal, with access to your files and shell. Install it, try it on a real project, and you'll see immediately why this is different from a chat interface.
