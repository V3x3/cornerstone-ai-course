## What is a hallucination?

When an AI generates confident, plausible-sounding information that is factually wrong, we call it a hallucination. The model isn't lying — it has no concept of truth. It's generating the most statistically likely continuation, and sometimes that continuation is incorrect.

## Why hallucinations happen

Three main causes:

1. **Training data gaps** — If the model was trained on limited information about a topic, it fills gaps with plausible-sounding text.
2. **Recency** — Models have a knowledge cutoff. Events after that date simply don't exist in their world.
3. **Specificity** — The more specific the claim (a precise date, a specific statistic, a legal citation), the higher the hallucination risk.

## What is bias?

Models learn from human-generated text. Human text reflects human biases — cultural, political, demographic. A model trained on internet text will encode the perspectives that dominate that text.

This doesn't make the model useless. It means you need to:
- Recognise whose perspective might be missing
- Probe alternative framings when the stakes are high
- Not treat the first answer as the only answer

## The limits that matter most

| Limit | What it means |
|-------|---------------|
| No real-time knowledge | Can't tell you what happened yesterday |
| No verification | Will confidently cite sources that don't exist |
| Context window | Can only "remember" what's in the current conversation |
| No persistent memory | Each conversation starts fresh (unless explicitly built otherwise) |

## How to use this

Don't avoid AI because it can hallucinate. Verify high-stakes facts. Use it for drafting, exploration, and synthesis — not as a primary source for citations you'll publish.

## Key takeaway

Hallucinations aren't bugs — they're a fundamental feature of how prediction-based models work. Know the failure modes, and you'll use these tools safely and effectively.
